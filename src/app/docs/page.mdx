# Introduction

NeuroLab combines clinical workflow software, an AI analysis stack, and user-facing applications into one repository. This documentation site is intended to explain the parts of that system that are active, documented, and reviewable from code today.

<Callout
  title='Documentation Scope'
  icon='◆'
>
  This site is intentionally scoped to the documented runtime, not the full set
  of ideas or experiments that may exist elsewhere in the repository. If a page
  and the code disagree, treat the codebase as the source of truth.
</Callout>

## Core mental states

The current documented analysis flows revolve around three recurring states:

| State | Typical meaning |
| :--- | :--- |
| **Relaxed** | calm or low-strain neural state |
| **Focused** | engaged or task-oriented state |
| **Stressed** | elevated strain, agitation, or overload signal |

These labels appear across both documentation and the current analysis stories, but you should still validate exact endpoint outputs against the active backend or AI service before treating them as a stable external contract.
